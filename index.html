Features

Built around trust, not just speed

The things that matter when you're letting something else touch your code: what it plans to do, what it costs, and how it fits the way you already work.

Plans before it acts

Before editing a file, Clew lays out its plan in plain language, then checks off each step as it happens. You always know what's about to change.

Any model, one interface

Cloud providers or models running on your own machine through Ollama and LM Studio. Switch which one is active without leaving the project.

Budget guardrails built in

Token usage, cost per provider, and burn rate, tracked live. Set a monthly limit and see exactly where you stand — no surprise bills.

Reads your project first

Clew looks at your actual file tree and existing code before making changes, instead of guessing at conventions from a single open file.

Make it yours

Eight built-in themes, from a warm charcoal-and-clay look to bold vivid gradients, plus adjustable text size for long sessions.

Free and open, on purpose

Apache 2.0 licensed. The code, the roadmap, and the bugs are all public — and contributions are genuinely welcome.

Inside Clew

What it actually looks like

No mockups — these are real screens from the app.

Transparency

The plan comes before the diff

Ask for something and Clew writes out its plan first — the files it'll look at, the edge cases it's accounting for, the tests it intends to run — then works through it step by step. It's the difference between an agent you supervise and one you just hope did the right thing.

Cost control

Know what you're spending, as you spend it

A live view of total tokens, estimated cost, and burn rate, split out by provider. Set a monthly budget once and Clew tracks your usage against it automatically — useful the moment you start mixing paid APIs with free-tier or local models.

Flexibility

One settings screen, every provider

Every provider gets the same layout: model name, API key, a test button, and a switch to make it the active one. Cloud and local providers sit side by side, so trying a new model is a config change, not a migration.

Personalization

Eight themes, one of which shaped this page

Ember — the warm charcoal-and-clay theme shown here — is also where this site got its palette. If dark-and-warm isn't your thing, Aurora leans into vivid gradients, Clay stays bright and cream-toned, and System just follows your OS.

Context

It sees the project, not just the prompt

A file browser sits alongside the chat so you can see exactly what Clew has access to. It reads real project structure before proposing changes, rather than working blind from whatever you happened to paste in.
